Introduction.

C is a powerful procedural-based programming language developed in 1972 by Dennis Ritchie within the halls of Bell Telephone Laboratories.

The C programming language was originally developed for use with the UNIX platform and has since spread to many other systems and applications. C has influenced a number of other programming languages, including C++ and Java.

Beginning programmers, especially those enrolled in computer science and engineering majors, need to build a solid foundation of operating systems, hardware, and application development concepts.

Numerous learning institutions accomplish this by teaching their students how to program in C so that they may progress to advanced concepts and other languages built upon C.

Semantic Errors

  • These errors occurred during the execution phase.
  • For example, if you attempt division by zero, an appropriate message will be displayed. Such errors are usually called run-time errors.
  • This means the source program must be edited and recompiled, and the object program executed again.
  • The process of identifying and correcting or removing the
  • bugs is called debugging.

Logic Error

  • Even if the program has no compile errors or run-time errors, the program may not be correct.
  • For example, the output result could be incorrect. The source of this error may be the original algorithm design.
  • If this is the case, we must return to the algorithm design phase and modify it, change the source program and compile and execute once more.
  • In order to test the correctness of the program, the program should be executed many times with various inputs.
  • However, this is still not a guarantee that the program is correct, since it may fail on other input data.

THE if STATEMENT

 

THE if STATEMENT
The general form
In general, the syntax for the if statement is:
if ( this condition is true )
execute this statement ;
The keyword if instruct the compiler that what follows is a decision control instruction. The condition following the keyword if is always enclosed within a pair of parentheses. If the condition is true, then the statement is executed. And if the condition is not true then the statement is not executed; instead the program skips the statement. The condition is expressed using the C’s ‘relational’ operators. The relational operators is used to compare two values to see whether the condition are equal to each other, unequal, or whether one is greater than the other.

THE While STATEMENT

The general form


In general, the syntax for the while statement is:

while (condition)
{

}

With the while statement, the condition is first tested and if the condition is TRUE then the statement or statements within the loop will be executed and if the condition is first tested is FALSE then the loop will not be executed at all. The condition will be tested repeatedly until the condition is false.

THE for STATEMENT

The for statement encloses one or more statements that form the body of the loop.These statements in the loop are executed repeatedly a certain number of times.The number of loop repetitions could be controlled in the count expression.

 The general form
 In general, the syntax for the for statement is:
for (start expression ; test expression; count expression)
{
statements;
}
  
C evaluates the start expression before the loop begins.Typically, the start expression is an assignment statement (such as ctr=1;)but it can be legal expression to be specified.C looks at and evaluates the start expressions only once, at the top of the loop.

UNIT 3: MODULAR PROGRAMMING

 

Modular programming is a software design technique that increases the extent to which software is composed of separate modules. This technique enables separation of modules, thus improve maintainability by enforcing logical boundaries between components. With Modular Programming, instead of creating a monolithic application, several smaller modules are built and compiled separately. Modular programming is more reusable than a monolithic design due to separate modules may then be reused in other projects.

INTRODUCTION

  • Any C program contains at least one function which is main( ) and this is where the program execution always begins. There is no limit on the number of functions that might be present in a C program and it is called in the sequence specified by the function calls in main( ). There are basically two types of functions:

Library functions Ex. printf( ), scanf( ) etc.
User-defined functions Ex. argentina( ), brazil( ) etc.
REASONS OF USING FUNCTION

  • Writing functions avoids rewriting the same code over and over. By using functions, it becomes easier to write programs and keep track of what they are doing. If the operation of a program can be divided into separate activities, and each activity placed in a different function, then each could be written and checked more or less independently. Separating the code into modular functions also makes the program easier to design and understand.

  • A function has to be created before it can be use. The following are methods in declaring, defining and calling a function:
  • Function declaration - followed by a semicolon with data types declared

int argentina( float a, char b) ;

  • Function calling - followed by a semicolon

argentina( ) ;

  • Function definition - followed by a pair of braces

argentina( )
{statement 1 ;
statement 2 ;
statement 3 ;
}

A function can call itself in a process called ‘recursion’. A function can also be called from other function, but a function cannot be defined in another function

POINTERS

A pointer is a variable that points to a location in which data is stored in the computer memory as shown in Figure 3.6. Each memory cell in the computer has an address that can be used to access that location. So a pointer variable points to a memory location we can access and change the contents of this memory location via the pointer.

 

 

 

Previous Next

Pointer Declarations

Declaration of pointer is in 3 steps:

  1. Specify the type of data stored in the location identified by the pointer.
  2. The asterisk tells the compiler this is a pointer variable.
  3. Give the name of the variable.

Example are as follows:
Declaring the pointer variable: type * variable name
int *ptr;
float *string;
The data pointed by the pointer is determined in the declaration of the pointer. For example:

  1. int *alpha ; //contain the address of an integer value
  2. char *ch ; // contain the address of a char value
  3. float *s ; // contain the address of a float value

Arrays

 

The C language allows the user to define a set of ordered data items known as an array. Suppose there are a set of grades that we wish to read into the computer and we wish to perform some operations on these grades, we cannot perform such an operation until each and every grade has been entered. It is quite tedious to declare each and every student grade as a variable especially since there may be a very large number. In C we can define a variable called grade, which represents not a single value of grade but an entire set of grades. Each element of the set can then be referenced by means of a number called as index number or subscript.

Simple Array

  • Simple array consists of one dimension. Example:
  • speed[ ] = {50,110,160,80,90}

Why Use Structures ?

  • We have seen earlier how ordinary variables can hold one piece of information and how arrays can hold a number of pieces of information of the same data type.
  • These two data types can handle a great variety of situations. But quite often we deal with entities that are collection of dissimilar data types.
  • We would be required to use a number of arrays, if we also decide to store name of the student, date of enroll, etc. To solve this problem, C provides a special data typethe structure.
  • A structure contains a number of data types grouped together.
  • These data types may or may not be of the same type.
